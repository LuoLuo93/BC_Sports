# 定时任务企微群推送功能配置说明

## 功能概述

定时任务执行后，根据配置的推送策略（`notifyStrategy`）自动发送通知到企业微信群机器人。

## 推送策略说明

| 策略值 | 说明 |
|--------|------|
| `ALWAYS` | 每次执行都推送（成功或失败） |
| `FAIL_ONLY` | 仅在任务失败时推送 |
| `DISABLED` | 不推送 |
| 空/未选择 | 不推送（默认） |

### 2. 配置企微群机器人 Webhook URL

1. 登录系统管理后台
2. 进入 **系统管理** → **系统配置**
3. 找到配置项 `schedule.notify.webhookUrl`
4. 填入企业微信群机器人的 Webhook URL

**获取 Webhook URL 的方法：**
1. 在企业微信中打开目标群聊
2. 点击群聊右上角「...」→「群机器人」→「添加」
3. 创建机器人后，复制 Webhook 地址（格式：`https://qyapi.weixin.qq.com/cgi-bin/webhook/send?key=xxx`）

### 3. 设置任务推送策略

1. 进入 **运维监控** → **定时任务**
2. 新增或编辑任务时，在「推送策略」下拉框中选择：
   - **总是推送** - 任务执行成功或失败都通知
   - **仅失败推送** - 只在任务失败时通知
   - **不推送** - 不发送通知

## 注意事项

1. **Webhook URL 必须配置** - 未配置时会跳过推送，不会报错
2. **消息长度限制** - 错误信息超过 500 字符会自动截断
3. **推送失败不影响任务** - 推送失败只记录警告日志，不影响任务执行结果
4. **并发安全** - 推送逻辑在任务执行完成后的 finally 块中，异步处理不会阻塞

## 测试建议

1. 先配置一个测试群的 Webhook URL
2. 创建一个测试任务，设置推送策略为「总是推送」
3. 手动执行任务，验证是否收到群消息
4. 故意制造一个失败场景（如配置错误的参数），验证失败通知是否正常
