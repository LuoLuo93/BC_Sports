## 二、CSRF 防护

### 发现的问题

| 优先级 | 问题 | 状态 |
|--------|------|------|
| 🔴 CRITICAL | Session 类型不匹配（Shiro vs Jakarta） | ✅ 已修复 |
| 🔴 CRITICAL | 无 Token 时静默跳过 | ✅ 已修复 |
| 🔴 CRITICAL | await 在非 async 函数中 | ✅ 已修复 |
| 🟠 HIGH | localStorage 存储 Token 不安全 | ✅ 已修复 |
| 🟡 MEDIUM | /doLogout CSRF 豁免 | ✅ 已修复 |

### 修复方案

1. **统一 Session 类型**：使用 HttpServletRequest.getSession() 替代 Shiro Session
2. **移除静默跳过**：Session 中有 Token 时必须验证
3. **修复 async 语法**：错误处理函数添加 async 关键字
4. **移除 localStorage**：Token 仅存储在内存变量中
5. **移除 /doLogout 豁免**：登出也需要 CSRF Token
