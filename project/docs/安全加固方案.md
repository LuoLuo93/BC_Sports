## 二、高危问题修复方案

### 2.1 XSS 防护

**问题**：用户输入直接存入数据库，无过滤

**方案**：添加 XSS 过滤器

```java
// 1. 创建 XssFilter.java
@WebFilter("/*")
public class XssFilter implements Filter {
    @Override
    public void doFilter(ServletRequest request, ServletResponse response, FilterChain chain)
            throws IOException, ServletException {
        chain.doFilter(new XssHttpServletRequestWrapper((HttpServletRequest) request), response);
    }
}

// 2. 创建 XssHttpServletRequestWrapper.java
public class XssHttpServletRequestWrapper extends HttpServletRequestWrapper {
    @Override
    public String getParameter(String name) {
        String value = super.getParameter(name);
        return value != null ? cleanXss(value) : null;
    }

    @Override
    public String[] getParameterValues(String name) {
        String[] values = super.getParameterValues(name);
        if (values == null) return null;
        return Arrays.stream(values).map(this::cleanXss).toArray(String[]::new);
    }

    private String cleanXss(String value) {
        // 移除危险标签和属性
        value = value.replaceAll("<script[^>]*>.*?</script>", "");
        value = value.replaceAll("<iframe[^>]*>.*?</iframe>", "");
        value = value.replaceAll("on\\w+\\s*=", "");
        value = value.replaceAll("javascript:", "");
        return HtmlUtils.htmlEscape(value);
    }
}
```

**优先级**：🔴 高

### 2.2 CSRF 防护

**问题**：无 CSRF Token 机制

**方案**：添加 SameSite Cookie + CSRF Token

```java
// 1. ShiroConfig.java 中添加 SameSite
@Bean
public SimpleCookie sessionIdCookie() {
    SimpleCookie cookie = new SimpleCookie("JSESSIONID");
    cookie.setHttpOnly(true);
    cookie.setSecure(true);
    cookie.setMaxAge(-1);
    cookie.setAttribute("SameSite", "Strict");  // 添加 SameSite
    return cookie;
}

// 2. 创建 CsrfFilter.java
public class CsrfFilter extends OncePerRequestFilter {
    @Override
    protected void doFilterInternal(HttpServletRequest request,
                                    HttpServletResponse response,
                                    FilterChain filterChain) throws ServletException, IOException {
        // GET/HEAD/OPTIONS 请求跳过
        String method = request.getMethod();
        if ("GET".equals(method) || "HEAD".equals(method) || "OPTIONS".equals(method)) {
            filterChain.doFilter(request, response);
            return;
        }

        // 检查 CSRF Token
        String token = request.getHeader("X-CSRF-Token");
        String sessionToken = (String) request.getSession().getAttribute("CSRF_TOKEN");

        if (token == null || !token.equals(sessionToken)) {
            response.setStatus(HttpServletResponse.SC_FORBIDDEN);
            response.getWriter().write("{\"code\":403,\"message\":\"CSRF token invalid\"}");
            return;
        }

        filterChain.doFilter(request, response);
    }
}

// 3. 登录时生成 Token
@GetMapping("/csrf")
public Result<String> getCsrfToken(HttpSession session) {
    String token = UUID.randomUUID().toString();
    session.setAttribute("CSRF_TOKEN", token);
    return Result.success(token);
}
```

**前端配合**：
```javascript
// request.js 中添加拦截器
request.interceptors.request.use(config => {
  const csrfToken = sessionStorage.getItem('csrfToken')
  if (csrfToken) {
    config.headers['X-CSRF-Token'] = csrfToken
  }
  return config
})
```

**优先级**：🔴 高

### 2.3 密码加密升级 (MD5 → BCrypt)

**问题**：MD5 算法已破解，不适用于密码存储

**方案**：迁移到 BCrypt

```java
// 1. 添加依赖 (pom.xml)
<dependency>
    <groupId>org.springframework.security</groupId>
    <artifactId>spring-security-crypto</artifactId>
</dependency>

// 2. 创建 BCryptPasswordUtil.java
public class BCryptPasswordUtil {
    private static final BCryptPasswordEncoder encoder = new BCryptPasswordEncoder(12);

    public static String encrypt(String rawPassword) {
        return encoder.encode(rawPassword);
    }

    public static boolean matches(String rawPassword, String encodedPassword) {
        return encoder.matches(rawPassword, encodedPassword);
    }
}

// 3. 创建 BCryptCredentialsMatcher.java
public class BCryptCredentialsMatcher implements CredentialsMatcher {
    @Override
    public boolean doCredentialsMatch(AuthenticationToken token, AuthenticationInfo info) {
        String submittedPassword = new String((char[]) token.getCredentials());
        String storedPassword = info.getCredentials().toString();
        return BCryptPasswordUtil.matches(submittedPassword, storedPassword);
    }
}

// 4. ShiroConfig 中替换
@Bean
public CredentialsMatcher credentialsMatcher() {
    return new BCryptCredentialsMatcher();  // 替换 HashedCredentialsMatcher
}

// 5. 数据库迁移脚本
-- 创建新字段
ALTER TABLE bc_sports_sys_user ADD password_new VARCHAR(100);

-- 迁移数据（需要应用层执行，因为需要重新加密）
-- 用户下次登录时自动迁移
```

**迁移策略**：
```java
// UserRealm.java 中添加迁移逻辑
@Override
protected AuthenticationInfo doGetAuthenticationInfo(AuthenticationToken token) {
    String username = (String) token.getPrincipal();
    User user = userMapper.selectByUsername(username);

    if (user != null) {
        // 检查是否需要迁移
        if (user.getPasswordNew() == null && user.getPassword() != null) {
            // 旧密码格式，验证后迁移
            String submittedPassword = new String((char[]) token.getCredentials());
            if (verifyOldPassword(submittedPassword, user.getPassword(), user.getSalt())) {
                // 迁移到 BCrypt
                String newEncoded = BCryptPasswordUtil.encrypt(submittedPassword);
                user.setPasswordNew(newEncoded);
                userMapper.updateById(user);
            }
        }
    }
    // ...
}
```

**优先级**：🔴 高

### 2.4 敏感信息加密存储

**问题**：数据库密码、API Secret 明文存储在 application.yml

**方案**：使用 Jasypt 加密

```xml
<!-- 1. 添加依赖 -->
<dependency>
    <groupId>com.github.ulisesbocchio</groupId>
    <artifactId>jasypt-spring-boot-starter</artifactId>
    <version>3.0.5</version>
</dependency>
```

```yaml
# 2. application.yml 配置
jasypt:
  encryptor:
    password: ${JASYPT_PASSWORD}  # 从环境变量读取
    algorithm: PBEWithMD5AndDES
```

```bash
# 3. 加密敏感信息
java -cp jasypt-1.9.3.jar org.jasypt.intf.cli.JasyptPBEStringEncryptionCLI \
  input="123456" password="your-secret-key" algorithm=PBEWithMD5AndDES

# 输出：ENC(加密后的字符串)
```

```yaml
# 4. 替换明文密码
spring:
  datasource:
    password: ENC(加密后的字符串)
```

### 3.2 文件上传安全加固

**问题**：仅检查 Content-Type，未校验文件内容

**方案**：多重校验

```java
public class FileUploadUtil {
    // 允许的文件扩展名
    private static final Set<String> ALLOWED_EXTENSIONS = Set.of(".jpg", ".jpeg", ".png", ".gif", ".svg");

    // 文件魔数校验
    private static final Map<String, byte[]> FILE_SIGNATURES = Map.of(
        "jpg", new byte[]{(byte) 0xFF, (byte) 0xD8, (byte) 0xFF},
        "png", new byte[]{(byte) 0x89, 0x50, 0x4E, 0x47},
        "gif", new byte[]{0x47, 0x49, 0x46, 0x38}
    );

    public static boolean validateFile(MultipartFile file) {
        // 1. 检查文件扩展名
        String originalName = file.getOriginalFilename();
        String ext = getExtension(originalName).toLowerCase();
        if (!ALLOWED_EXTENSIONS.contains(ext)) {
            return false;
        }

        // 2. 检查 Content-Type
        String contentType = file.getContentType();
        if (contentType == null || !contentType.startsWith("image/")) {
            return false;
        }

        // 3. 检查文件魔数
        try {
            byte[] header = new byte[8];
            file.getInputStream().read(header);
            if (!validateMagicBytes(header, ext)) {
                return false;
            }
        } catch (IOException e) {
            return false;
        }

        // 4. 检查文件大小
        if (file.getSize() > 10 * 1024 * 1024) {  // 10MB
            return false;
        }

        // 5. 生成安全的文件名
        // 避免路径穿越攻击
        String safeName = UUID.randomUUID() + ext;

        return true;
    }

    private static String getExtension(String filename) {
        int dotIndex = filename.lastIndexOf('.');
        return dotIndex > 0 ? filename.substring(dotIndex) : "";
    }

    private static boolean validateMagicBytes(byte[] header, String ext) {
        byte[] signature = FILE_SIGNATURES.get(ext.substring(1));
        if (signature == null) return true;  // 未知类型跳过

        for (int i = 0; i < signature.length; i++) {
            if (header[i] != signature[i]) {
                return false;
            }
        }
        return true;
    }
}
```

**优先级**：🟡 中

### 3.3 CORS 生产环境配置

**问题**：生产环境无 CORS 配置

**方案**：添加生产环境 CORS

```java
@Configuration
@Profile("prod")
public class ProdCorsConfig implements WebMvcConfigurer {
    @Override
    public void addCorsMappings(CorsRegistry registry) {
        registry.addMapping("/api/**")
                .allowedOrigins("https://your-domain.com")  // 生产域名
                .allowedMethods("GET", "POST", "PUT", "DELETE")
                .allowedHeaders("Content-Type", "Authorization", "X-CSRF-Token")
                .allowCredentials(true)
                .maxAge(3600);
    }
}
```

**优先级**：🟡 中
