# 定时任务企微推送功能更新

## 更新日期
2026-06-05

## 功能概述

为定时任务模块新增企业微信群机器人推送功能，任务执行后可自动发送通知到企微群。

## 部署步骤

### 1. 执行数据库脚本

```sql
-- 1. 给定时任务表添加字段
ALTER TABLE bc_sports_sys_schedule_job ADD (
    notify_strategy VARCHAR2(20),
    PARAMS VARCHAR2(2000)
);

COMMENT ON COLUMN bc_sports_sys_schedule_job.notify_strategy IS '推送策略(ALWAYS/FAIL_ONLY/DISABLED)';
COMMENT ON COLUMN bc_sports_sys_schedule_job.PARAMS IS '任务参数(JSON)';

-- 2. 添加系统配置项
INSERT INTO bc_sports_sys_config (id, config_key, config_value, config_name, config_group, sort, remark, create_time, create_by)
VALUES (rawtohex(sys_guid()), 'schedule.notify.webhookUrl', '', '企微群机器人Webhook', 'schedule', 1, '企业微信群机器人的webhook URL', SYSTIMESTAMP, 'system');

INSERT INTO bc_sports_sys_config (id, config_key, config_value, config_name, config_group, sort, remark, create_time, create_by)
VALUES (rawtohex(sys_guid()), 'schedule.notify.defaultStrategy', 'FAIL_ONLY', '默认推送策略', 'schedule', 2, '新建定时任务时的默认推送策略', SYSTIMESTAMP, 'system');

COMMIT;
```

### 2. 重新编译部署

```bash
# 后端
cd project
mvn clean package -DskipTests

# 前端
cd frontend
npm run build
```

### 3. 配置 Webhook URL

1. 登录系统
2. 进入 **系统设置** → **通知配置**
3. 输入企微群机器人的 Webhook URL
4. 点击「测试」按钮验证连接
5. 点击「保存设置」

### 4. 设置任务推送策略

1. 进入 **运维监控** → **定时任务**
2. 新增或编辑任务
3. 选择推送策略：
   - **总是推送** - 每次执行都通知
   - **仅失败推送** - 只在失败时通知（推荐）
   - **不推送** - 不发送通知

## 功能特性

### 推送策略

| 策略 | 说明 |
|------|------|
| `ALWAYS` | 每次执行都推送（成功或失败） |
| `FAIL_ONLY` | 仅在任务失败时推送 |
| `DISABLED` | 不推送 |
| 空/未选择 | 不推送 |

### 推送消息格式

使用 Markdown 格式，包含以下信息：
- 任务名称
- 执行状态（成功/失败）
- 触发类型（CRON定时/手动触发）
- 开始时间
- 结束时间
- 执行耗时
- 错误信息（仅失败时）

### 消息示例

**任务成功通知：**
```
### 定时任务执行通知
> 任务名称：iHR员工同步
> 执行状态：✅ 成功
> 触发类型：CRON定时
> 开始时间：2026-06-05 10:00:00
> 结束时间：2026-06-05 10:00:02
> 执行耗时：2.3s
```

**任务失败通知：**
```
### 定时任务执行通知
> 任务名称：南讯CRM会员标签推送
> 执行状态：❌ 失败
> 触发类型：手动触发
> 开始时间：2026-06-05 10:05:00
> 结束时间：2026-06-05 10:05:45
> 执行耗时：45.2s
> 错误信息：API调用失败...
```

### 测试功能

在系统设置页面可以测试 Webhook 连接，发送测试消息到企微群验证配置是否正确。

## 注意事项

1. **Webhook URL 必须配置** - 未配置时会跳过推送，不会报错
2. **消息长度限制** - 错误信息超过 500 字符会自动截断
3. **推送失败不影响任务** - 推送失败只记录警告日志
4. **并发安全** - 推送在任务执行完成后的 finally 块中执行
