# 密码加密升级说明 (MD5 → BCrypt)

## 概述

项目已将密码加密算法从 MD5 升级到 BCrypt，采用渐进式迁移策略，无需强制用户重置密码。

## 技术方案

### 算法对比

| 特性 | MD5 | BCrypt |
|------|-----|--------|
| 安全性 | 已破解，不安全 | 目前安全 |
| 速度 | 极快（易暴力破解） | 慢（防暴力破解） |
| 盐值 | 需要手动管理 | 自动内置 |
| 迭代次数 | 2次（太少） | 12次（可配置） |

### 迁移策略

采用**渐进式迁移**，无需强制重置密码：

```
新用户注册 → 直接使用 BCrypt 加密
    ↓
老用户登录 → 检测到旧 MD5 密码
    ↓
验证通过 → 自动迁移为 BCrypt
    ↓
后续登录 → 使用 BCrypt 验证
```

## 工作流程

### 1. 新用户注册

```
addUser()
    ↓
BCryptPasswordUtil.encrypt(password)
    ↓
存储到 password 和 passwordNew 字段
```

### 2. 老用户登录

```
doGetAuthenticationInfo()
    ↓
检查 passwordNew 字段
    ↓
┌─── 有值且是 BCrypt 格式 ──────────────────────┐
│   使用 BCryptPasswordUtil.matches() 验证       │
│   验证通过 → 登录成功                          │
│   验证失败 → 返回错误                          │
└──────────────────────────────────────────────┘
┌─── 无值或不是 BCrypt 格式 ─────────────────────┐
│   使用旧的 MD5 方式验证                        │
│   验证通过 → 自动迁移为 BCrypt                 │
│   验证失败 → 返回错误                          │
└──────────────────────────────────────────────┘
```

### 3. 密码迁移

```
MD5 验证通过
    ↓
BCryptPasswordUtil.encrypt(password)
    ↓
调用 migratePassword(userId, newBCryptPassword)
    ↓
更新 passwordNew 字段
    ↓
后续登录使用 BCrypt 验证
```

## 数据库变更

### 新增字段


### 字段说明

| 字段 | 类型 | 说明 |
|------|------|------|

## 安全性提升

### MD5 的问题

- 计算速度极快，易被暴力破解
- 已有彩虹表攻击
- 仅 2 次迭代，安全性低

### BCrypt 的优势

- 计算速度慢，防暴力破解
- 内置盐值，防彩虹表攻击
- 12 次迭代，安全性高
- 自动处理盐值生成和存储

## 注意事项

1. **渐进式迁移**：老用户无需重置密码，登录时自动迁移
2. **向后兼容**：旧的 password 和 salt 字段保留，支持未迁移的用户
3. **性能影响**：BCrypt 计算较慢，但对用户体验影响可忽略
4. **日志记录**：迁移过程会记录日志，便于监控

## 后续优化

1. **清理旧字段**：所有用户迁移完成后，可考虑清理 password 和 salt 字段
2. **强制迁移**：设置截止日期，强制未迁移用户重置密码
3. **密码策略**：配合 BCrypt 实施更强的密码策略
