# 推送服务扩展指南

## 架构概述

推送服务采用**策略模式**设计，支持多渠道推送，可轻松扩展新的推送方式。

```
┌─────────────────────────────────────────────────────────────┐
│                      NotifyManager                          │
│                    (推送管理器 - 统一入口)                    │
├─────────────────────────────────────────────────────────────┤
│  List<NotifyChannel> channels                               │
│  ├── WechatChannel (企微群机器人)                            │
│  ├── DingTalkChannel (钉钉机器人) [可扩展]                   │
│  ├── EmailChannel (邮件通知) [可扩展]                        │
│  └── SmsChannel (短信通知) [可扩展]                          │
└─────────────────────────────────────────────────────────────┘
```

## 核心接口

### NotifyChannel 接口

```java
public interface NotifyChannel {
    String getName();                    // 渠道名称
    boolean isEnabled();                 // 是否启用
    void send(NotifyMessage message);    // 发送消息
}
```

### NotifyMessage 消息模型

```java
public class NotifyMessage {
    private String title;           // 消息标题
    private String content;         // 消息内容（通用格式）
    private NotifyType type;        // 消息类型
    private Map<String, Object> extras;  // 扩展参数
}
```

### NotifyType 消息类型

```java
public enum NotifyType {
    TASK_RESULT,    // 任务执行结果
    ALERT,          // 告警通知
    INFO            // 普通信息
}
```

## 扩展新渠道

### 步骤 1：创建渠道实现类

以钉钉机器人为例，创建 `DingTalkChannel.java`：

```java
package com.bcsport.admin.service.notify.channel;

import com.bcsport.admin.service.ConfigService;
import com.bcsport.admin.service.notify.NotifyChannel;
import com.bcsport.admin.service.notify.NotifyMessage;
import lombok.extern.slf4j.Slf4j;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.stereotype.Component;

@Slf4j
@Component
public class DingTalkChannel implements NotifyChannel {

    private static final String CONFIG_KEY_WEBHOOK = "schedule.notify.dingtalk.webhook";

    @Autowired
    private ConfigService configService;

    @Override
    public String getName() {
        return "钉钉机器人";
    }

    @Override
    public boolean isEnabled() {
        String webhook = configService.getString(CONFIG_KEY_WEBHOOK);
        return webhook != null && !webhook.isBlank();
    }

    @Override
    public void send(NotifyMessage message) {
        String webhook = configService.getString(CONFIG_KEY_WEBHOOK);
        if (webhook == null || webhook.isBlank()) {
            log.debug("钉钉webhook未配置，跳过推送");
            return;
        }

        // 转换为钉钉消息格式并发送
        // ...
    }
}
```

### 步骤 2：添加配置项

在数据库中添加配置项：

```sql
INSERT INTO bc_sports_sys_config (id, config_key, config_value, config_name, config_group, sort, remark, create_time, create_by)
VALUES (rawtohex(sys_guid()), 'schedule.notify.dingtalk.webhook', '', '钉钉机器人Webhook', 'schedule', 3, '钉钉群机器人的webhook URL', SYSTIMESTAMP, 'system');
```

### 步骤 3：完成

无需修改其他代码，Spring 会自动将 `DingTalkChannel` 注入到 `NotifyManager` 的 `channels` 列表中。

## 使用方式

### 发送任务执行结果

```java
@Autowired
private NotifyManager notifyManager;

// 发送任务执行结果（自动发送到所有启用的渠道）
notifyManager.sendTaskResult(
    "iHR员工同步",           // 任务名称
    1,                       // 状态 (1=成功, 0=失败)
    "CRON",                  // 触发类型
    startTime,               // 开始时间
    endTime,                 // 结束时间
    2300,                    // 耗时(毫秒)
    null                     // 错误信息
);
```

### 发送自定义消息

```java
NotifyMessage message = NotifyMessage.builder()
    .title("系统告警")
    .content("磁盘使用率超过90%")
    .type(NotifyType.ALERT)
    .extras(Map.of("server", "prod-01", "diskUsage", "92%"))
    .build();

notifyManager.send(message);
```

### 发送测试消息

```java
notifyManager.sendTest();
```

## 配置管理

### 系统设置页面

在 `Settings.vue` 的"通知配置" Tab 中，可以配置各渠道的参数。

### 前端扩展

如需在前端添加新渠道的配置项：

1. 在 `Settings.vue` 的"通知配置" Tab 中添加表单项
2. 配置项的 `configKey` 与后端的 `CONFIG_KEY_*` 常量对应
3. 保存时会自动更新 `bc_sports_sys_config` 表

## 重试机制

推送服务内置重试机制，网络请求失败时自动重试：

| 参数 | 默认值 | 说明 |
|------|--------|------|
| 最大重试次数 | 3 | 最多重试 3 次 |
| 基础重试间隔 | 1000ms | 第一次重试等待 1 秒 |
| 退避策略 | 指数退避 + 随机抖动 | 避免重试风暴 |

**可重试异常**：
- 网络连接异常（ConnectException、SocketTimeoutException 等）
- HTTP 5xx 错误（服务器临时错误）
- 超时异常

**不可重试异常**：
- 配置错误
- HTTP 4xx 错误（客户端错误）
- 业务逻辑错误

使用示例：
```java
RetryUtil.executeWithRetryVoid(() -> {
    // 你的网络请求代码
}, 3, 1000);
```

详见 `docs/推送重试机制说明.md`

## 注意事项

1. **渠道自动发现**：Spring 会自动注入所有实现 `NotifyChannel` 接口的 Bean
2. **启用判断**：`isEnabled()` 返回 `false` 的渠道会被跳过
3. **异常隔离**：单个渠道发送失败不影响其他渠道
4. **消息格式**：各渠道自行将 `NotifyMessage` 转换为自己的格式
5. **配置缓存**：配置通过 `ConfigService` 缓存，修改后需调用 `reload()` 或重启生效
6. **重试幂等**：确保重试的操作是幂等的（重复执行不会产生副作用）
