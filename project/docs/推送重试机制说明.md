# 推送重试机制说明

## 概述

推送服务已集成重试机制，当网络请求失败时会自动重试，提高推送成功率。

## 重试策略

### 配置参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| 最大重试次数 | 3 | 最多重试 3 次 |
| 基础重试间隔 | 1000ms | 第一次重试等待 1 秒 |
| 最大重试间隔 | 30000ms | 单次重试最大等待 30 秒 |
| 退避策略 | 指数退避 + 随机抖动 | 避免重试风暴 |

### 退避算法

```
第 0 次重试：1s ± 25% (0.75s ~ 1.25s)
第 1 次重试：2s ± 25% (1.5s ~ 2.5s)
第 2 次重试：4s ± 25% (3s ~ 5s)
```

### 可重试异常

以下异常类型会触发重试：

1. **网络异常**
   - `ConnectException` - 连接被拒绝
   - `SocketTimeoutException` - 连接超时
   - `UnknownHostException` - DNS 解析失败
   - `IOException` - IO 异常

2. **特定错误消息**
   - 包含 "timeout" 或 "超时"
   - 包含 "connection" 或 "连接"
   - HTTP 5xx 错误（服务器临时错误）

### 不可重试异常

以下异常不会触发重试：

- 配置错误（Webhook URL 为空）
- URL 格式错误
- HTTP 4xx 错误（客户端错误）
- 企微业务错误（如 errcode != 0）

## 实现细节

### RetryUtil 工具类

```java
public class RetryUtil {

    /**
     * 执行带重试的操作（有返回值）
     */
    public static <T> T executeWithRetry(RetryableAction<T> action, int maxRetries, long baseDelay) throws Exception {
        Exception lastException = null;

        for (int attempt = 0; attempt <= maxRetries; attempt++) {
            try {
                return action.execute();
            } catch (Exception e) {
                lastException = e;

                // 判断是否可重试
                if (!isRetryable(e) || attempt >= maxRetries) {
                    throw e;
                }

                // 计算退避时间
                long delay = calculateDelay(attempt, baseDelay);
                log.warn("操作失败，第 {}/{} 次重试，等待 {}ms，原因: {}",
                        attempt + 1, maxRetries, delay, e.getMessage());

                Thread.sleep(delay);
            }
        }

        throw lastException;
    }

    /**
     * 判断异常是否可重试
     */
    private static boolean isRetryable(Exception e) {
        // 网络相关异常
        if (e instanceof ConnectException ||
            e instanceof SocketTimeoutException ||
            e instanceof UnknownHostException ||
            e instanceof IOException) {
            return true;
        }

        // 包含特定消息的异常
        String message = e.getMessage();
        if (message != null) {
            if (message.contains("timeout") || message.contains("超时")) {
                return true;
            }
            if (message.contains("connection") || message.contains("连接")) {
                return true;
            }
            if (message.contains("HTTP") && message.contains("5")) {
                return true;
            }
        }

        return false;
    }

    /**
     * 计算退避时间（指数退避 + 随机抖动）
     */
    private static long calculateDelay(int attempt, long baseDelay) {
        long exponentialDelay = baseDelay * (1L << attempt);
        double jitter = 0.75 + Math.random() * 0.5;
        long delay = (long) (exponentialDelay * jitter);
        return Math.min(delay, 30000);
    }
}
```

### WechatChannel 使用示例

```java
@Override
public void send(NotifyMessage message) {
    String webhookUrl = configService.getString(CONFIG_KEY_WEBHOOK_URL);
    try {
        String content = convertToMarkdown(message);
        // 使用重试机制发送消息
        RetryUtil.executeWithRetryVoid(
            () -> sendMarkdownMessage(webhookUrl, content),
            MAX_RETRIES,
            BASE_DELAY
        );
    } catch (Exception e) {
        log.error("企微群推送失败（已重试{}次）: {}", MAX_RETRIES, e.getMessage(), e);
        throw new RuntimeException("企微群推送失败: " + e.getMessage(), e);
    }
}
```

## 配置建议

### 生产环境

```java
// 默认配置即可
private static final int MAX_RETRIES = 3;
private static final long BASE_DELAY = 1000;
```

### 高可用场景

```java
// 增加重试次数和间隔
private static final int MAX_RETRIES = 5;
private static final long BASE_DELAY = 2000;
```

### 低延迟场景

```java
// 减少重试次数，快速失败
private static final int MAX_RETRIES = 1;
private static final long BASE_DELAY = 500;
```

## 注意事项

1. **幂等性**：确保重试的操作是幂等的（重复执行不会产生副作用）
2. **超时设置**：HTTP 请求的超时时间应小于重试间隔
3. **资源消耗**：重试会消耗额外的线程和网络资源
4. **监控告警**：重试失败后应记录指标并告警

## 后续优化

1. **配置化**：将重试参数提取到系统配置表
2. **熔断机制**：连续失败达到阈值时暂时停止重试
3. **异步重试**：使用消息队列进行异步重试
4. **重试统计**：记录重试次数和成功率，便于监控
